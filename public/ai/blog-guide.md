# 🧠 뇌 용량 확보용 - AI 블로그 글 작성 가이드

이 문서는 AI가 블로그 글을 작성할 때 참고하는 가이드입니다.

## 블로그 정보

- **이름**: 뇌 용량 확보용
- **컨셉**: 머릿속 비우고 RAM 확보하는 블로그
- **작성자**: 상원 (개발자, 풀스택, 블록체인, AI)
- **URL**: https://blog.sangwon0001.xyz

## 카테고리

현재 사용 중인 카테고리:
- `ai` - AI, LLM, 머신러닝 관련
- `tech` - 개발, 기술 일반

새 카테고리가 필요하면 `content/[새카테고리]/` 폴더를 만들면 자동 추가됨.

## 글 톤 & 스타일

### 말투
- **반말 사용** (편하게, 친근하게)
- 딱딱하지 않게, 대화하듯이
- 적당한 유머 OK
- 이모지 적당히 사용 가능

### 예시
```
❌ "오늘은 Next.js에 대해 알아보겠습니다."
✅ "Next.js 써봤는데 생각보다 괜찮더라."

❌ "이 기술의 장점은 다음과 같습니다."
✅ "장점? 일단 빠르고, 설정 귀찮은 거 알아서 해줌."
```

### 구조
1. 서론은 짧게 (왜 이 글을 쓰는지)
2. 본론은 실용적으로 (코드 예시, 실제 사용법)
3. 결론은 솔직하게 (좋은 점, 아쉬운 점)

## 파일 형식

### 위치
```
content/[카테고리]/[slug].md
```

### Frontmatter (필수)
```yaml
---
title: "글 제목"
description: "2-3문장 요약 (목록에 표시됨)"
date: "YYYY-MM-DD"
tags: ["태그1", "태그2", "태그3"]
---
```

### 예시
```yaml
---
title: "Next.js 15에서 달라진 것들"
description: "Next.js 15 써보니까 꽤 많이 바뀌었더라. 주요 변경사항이랑 마이그레이션 팁 정리해봄."
date: "2025-02-04"
tags: ["nextjs", "react", "frontend"]
---
```

## 마크다운 문법

### 지원하는 요소
- 제목: `#`, `##`, `###`, `####`
- 코드블록: \`\`\`언어 (typescript, javascript, python, bash, etc.)
- 인라인 코드: \`코드\`
- 인용: `>`
- 리스트: `-`, `1.`
- 링크: `[텍스트](URL)`
- 이미지: `![alt](/images/slug/filename.png)`
- 굵게: `**텍스트**`
- 기울임: `*텍스트*`

### 코드블록 예시
\`\`\`typescript
const greeting = "Hello, World!";
console.log(greeting);
\`\`\`

### 이미지
- 위치: `public/images/[slug]/` 폴더에 저장
- 참조: `![설명](/images/[slug]/파일명.png)`
- 자동으로 WebP/AVIF 변환 및 최적화됨

## 글 작성 템플릿

```markdown
---
title: "제목을 여기에"
description: "2-3문장으로 글 요약"
date: "2025-02-04"
tags: ["tag1", "tag2"]
---

## 들어가며

왜 이 글을 쓰게 됐는지 간단히.

## 본론 제목

핵심 내용.

### 소제목 1

내용...

\`\`\`typescript
// 코드 예시
\`\`\`

### 소제목 2

내용...

## 마무리

결론, 느낀 점, 추천 여부 등.
```

## 주의사항

1. **slug는 영어로** - 파일명은 영어 소문자와 하이픈만 (`my-first-post.md`)
2. **date 형식** - `YYYY-MM-DD` 형식 엄수
3. **이미지 경로** - 반드시 `/images/`로 시작
4. **코드 언어 명시** - 코드블록에 언어 꼭 적기

## AI 글 작성 요청 예시

사용자가 이렇게 요청할 수 있음:

> "Next.js App Router에 대한 글 써줘. 실제로 써본 경험 기반으로, 장단점 위주로."

이때 AI는:
1. 이 가이드의 톤/스타일 따르기
2. 템플릿 구조 사용
3. 실용적인 코드 예시 포함
4. frontmatter 완성해서 제공
