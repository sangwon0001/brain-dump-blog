## 들어가며

블로그 하나 만들려고 했다. 근데 요즘 AI 코딩 도구가 너무 좋아져서, 이번엔 **Claude Code**한테 거의 다 맡겨봤다. 결과? 생각보다 잘 됐다.

이 글은 "AI한테 블로그 만들어달라고 하면 어디까지 되나?" 실험 기록이다.

## 왜 이 스택을 선택했나

### Next.js + MDX + Vercel

```
나 → Claude Code → Next.js 블로그 → Vercel 배포
```

선택 이유:
- **Next.js**: SSG(정적 생성) 지원, SEO 좋음
- **MDX**: 마크다운으로 글 쓰기 편함, 코드 하이라이팅 쉬움
- **Vercel**: Next.js 만든 회사, 배포가 그냥 됨

근데 사실 제일 큰 이유는 **"나, AI, 프론트엔드 개발자 모두 이해하기 쉬운 구조"**였다.

### 마크다운 기반의 장점

```
content/
├── ai/
│   └── intro-to-llm.md
└── tech/
    └── getting-started-nextjs.md
```

이 구조면:
- **나**: 마크다운으로 글 쓰면 끝
- **AI**: 파일 구조 읽고 바로 글 생성 가능
- **개발자**: 복잡한 CMS 없이 git으로 관리

DB 붙이고 어드민 만들고... 그런 거 안 해도 된다.

## Claude Code 활용법

### 효과적인 프롬프트

처음엔 이렇게 시작했다:

> "Next.js로 블로그 만들어줘"

이것만으로도 기본 구조는 나온다. 근데 더 구체적으로 갈수록 결과가 좋아졌다.

**잘 먹히는 프롬프트 패턴:**

```
[현재 상황] + [원하는 것] + [제약 조건]
```

예시:
> "지금 이 블로그의 테마 색을 바꾸자. 핫핑크로. 다크모드 배경도 어울리게."

> "SEO 작업 들어갈까? sitemap, robots.txt, OG 태그 다 해줘."

### 대화하듯이 진행

Claude Code의 좋은 점은 **맥락을 기억한다**는 거다.

```
나: "테마 색 바꾸자"
Claude: (변경함)
나: "다크모드 배경도 어울리게"
Claude: (다크모드도 수정)
나: "좋아 커밋해"
Claude: (커밋함)
```

한 세션에서 계속 이어가니까 일일이 설명 안 해도 됐다.

### 모르는 건 물어보게 하기

```
나: "바이오도 넣어줘"
Claude: "어떤 정보 넣을까요?"
나: "github cli로 조회해봐"
Claude: (gh api user 실행 → 정보 추출 → 바이오 생성)
```

AI가 알아서 하게 두되, 필요한 정보는 물어보게 했다.

## 폴더 구조

최종 구조:

```
medium-blogging/
├── content/           # 블로그 글 (마크다운)
│   ├── ai/
│   └── tech/
├── public/
│   ├── images/        # 포스트 이미지
│   └── ai/            # AI 글쓰기 가이드
├── src/
│   ├── app/           # Next.js App Router
│   ├── components/    # React 컴포넌트
│   └── lib/           # 유틸리티 (MDX 파싱)
├── CLAUDE.md          # AI 어시스턴트 가이드
└── README.md
```

핵심은 `content/` 폴더다. 여기에 마크다운 파일만 넣으면 자동으로 블로그 글이 된다.

### CLAUDE.md

이건 좀 꿀팁인데, 프로젝트 루트에 `CLAUDE.md` 파일을 두면 Claude Code가 자동으로 읽는다.

```markdown
# CLAUDE.md

## Commands
npm run dev      # 개발 서버
npm run build    # 빌드

## 새 글 추가하기
content/[category]/[slug].md 에 파일 생성
```

이렇게 해두면 매번 설명 안 해도 Claude가 알아서 참고한다.
